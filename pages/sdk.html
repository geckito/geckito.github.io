The Geckito SDK

The Geckito SDK provides all build tooling and the templates for building your device image. This includes a native compiler toolchain, a cross compiler and build tooling for package and image building (pbuild).

The SDK is delivered in multiple forms:

Linux Container

A container is provided which includes the entire tooling. Further software can get installed via zypper inside of the container. It can get pulled on any Linux distribution using podman or docker tooling.

$ podman pull registry.opensuse.org/geckito/sdk/main/images/geckito/sdk:15.3

To run a shell execute

$ docker run -ti $IMAGE_ID

The container can be used in Eclipse IDE. Install for that the "C/C++ Docker Container Launch Support" via Help - "Install new software" menu. Afterwards you need to select the container via Project - Properties - C/C++ Build - Settings - Container Settings. The "build in container" check box has to be ticked and select the Geckito SDK container for building.

The container can be used in MS Visual Code via the Remote - Containers extension.

MS-Windows WSL Image

A MS-Windows WSL image is provided which includes the same tooling. Further software can get installed via zypper inside of the container. The image is currently not yet official signed and needs therefore installed as documented here. The WSL image can get found here.

Plain package installation

openSUSE Linux or SUSE Linux Enterprise users can also install the packages directly without a container. Find your matching repository here.
