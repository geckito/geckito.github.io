Embedded Linux with long-term maintenance support

Goals

The Geckito project has multiple sub projects. All of them have the goal to provide the best solution for maintainable IoT solutions.

IoT Ready Code Base

Enhance the existing code base of a classic Linux distribution to become ready for IoT appliances. This gives the advantage to have a maintained code base in your solution. openSUSE Tumbleweed is our starting point for a rolling release based distribution. Afterwards the openSUSE Leap code base will deliver a stable base for defined time. And an even longer timeframe is possible via the SUSE Linux Enterprise code base.

However, multiple projects need to be done to fullfill this goal.

Bit-wise reproducible builds

SUSE distributions are build in OBS which guarantees reproducible build environments. And build results are compared automatically with former build results. However, there are still some pieces where builds change on rebuild. For example when a time code is included. Overview page on openSUSE.

Cross Architecture Builds

Packages shall also become cross buildable as much as possible. This allows fast rebuilds avoiding any emulator.

  • Code Base cross buildable: 30%
  • Image cross buildable: 90%
openSUSE Leap: Single Layer Rebuilds

openSUSE Leap and SUSE Linux Enterprise distributions are using service pack layering to control changes due to changed binaries over the live time (up to 15 years). While this is good for keeping changes low it creates also the problem that each package needs to stay in it's build layer. And if it get moved to another layer build failures may occur (and of course extensive testing is needed). For IoT projects, where you may want to rebuild all sources this creates a complexity which is not wanted. Therefore the code base will get adapted that all sources are rebuildable on latest service packs and maintenance updates. The FrontRunner Project in OBS show the current state of problems to be solved.

Build Tooling

The new pbuild tool allows to build multiple packages in one go. It evaluates the need on rebuilds automatically, can build various formats and creates dynamic sysroots for cross builds. It can be used to rebuild everything from source or just the needed parts.

The tool is agnoistic to the way you manage your sources. It works with git clones or with any OBS project checkout.

SDK for Application Developers

Application developers need to be able to build their applications in an easy and fast way. It must be possible to build for the target from any IDE. Important subgoals are to support Eclipse, MS Visual Code and Qt Creator.

Device specific templates

We want to build up a device specific template collection to allow easy start for many devices. The first device which has been picked is the Raspberry Pi 4.
