Linux for Embedded, long term maintainable

What is Geckito?

Geckito is a tool set that makes it possible to create ready-to-use system images for your embedded, edge, and IoT projects. The resulting system images are based on openSUSE and SUSE, a high-quality, rock solid, and actively maintained Linux distribution. Geckito can run on Linux and Windows, and it provides support for multiple target architecture, including x86 and ARM. Using Geckito you can compile the required software from sources, or you can use existing binary packages from trusted repositories. Moreover, Geckito integrates nicely with your favorite IDE.

Find out more about Geckito's components below.

Build Environment

pbuild is a cornerstone of Geckito. This tool allows you to build images, containers, packages. While pbuild works fine offline, it can make use of online resources.

A Linux distribution in binary and source form

The Geckito project is based on the popular openSUSE distribution to deliver binaries and sources suitable for embedded development. It offers reproducible and cross architecture builds for a defined maintenance time frame.

The shared code base with the build script guarantees that the builds stay compatible with the build results from the Open Build Service (esp. for openSUSE and SUSE Linux Enterprise distributions).

A collection of templates

Templates demonstrate how the to build small, targeted images. This can include the usage of pre-build binaries, own rpm packages, container and own application builds.

What does Geckito offer?

Maintained and stable distributions

Part of the project is to enhance stable distributions with the needs for IoT. This includes an easy way to rebuild and modify central distribution parts, but keeping the possibility to reuse as much as possible from an enterprise ready distribution. This allows to guarantee maintenance time lines and reviewed licenses and sources.

SDK for application development

The simplest way is to build for the native distribution using the standard tools. This can be done via the SUSE build script, the Open Build Service or just native tooling.

MS-Windows users can use the SDK via provided WSL images, directly usable from our Eclipse or MS Visual Code IDE.

Cross Architecture Builds

Cross Architecture build tooling is also developed as part of the Geckito project. It is available for application development, but also supported for package or image builds allowing fast turn around times.
